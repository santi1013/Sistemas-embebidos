Detailed Description

Interfaz pública del módulo de registro en archivo.

Este archivo declara las funciones encargadas de inicializar el sistema de almacenamiento de registros en la microSD y de escribir datos formateados en el archivo de log del proyecto.

El módulo file_logger actúa como una capa intermedia entre las tareas que generan información y el driver de acceso a la tarjeta SD. Su objetivo es simplificar la escritura de registros y mantener separada la lógica de adquisición/procesamiento de la lógica de almacenamiento.

◆ file_logger_init()

esp_err_t file_logger_init ( void )

Inicializa el módulo de escritura en archivo.

Esta función prepara el archivo de log para comenzar a almacenar registros. Internamente verifica que la microSD esté montada, abre el archivo definido en la configuración del proyecto y, si corresponde, escribe el encabezado del archivo.

Debe ejecutarse antes de llamar a la función de escritura de registros.

Returns
  • ESP_OK si la inicialización fue exitosa.
  • ESP_FAIL si la SD no está montada, si el archivo no pudo abrirse o si falló la escritura del encabezado.

Inicializa el módulo de escritura en archivo.

Esta función verifica primero que la tarjeta microSD esté montada. Luego intenta abrir el archivo definido en la configuración del proyecto y, si aún no se ha hecho, escribe el encabezado del archivo CSV o TXT.

Su lógica es preparar el entorno de almacenamiento antes de que comiencen a llegar registros desde las tareas del sistema.

Returns
  • ESP_OK si el archivo quedó listo para escritura.
  • ESP_FAIL si la SD no está montada, si no se pudo abrir el archivo o si falló la escritura del encabezado.

esp_err_t file_logger_write_record ( const log_record_t * record)

Escribe un registro formateado en el archivo de log.

Esta función recibe una estructura de tipo log_record_t que contiene una línea de texto ya construida y lista para ser almacenada. El contenido es enviado al driver de la microSD para su escritura en el archivo.

Parameters
recordPuntero al registro que se desea guardar.
Returns
  • ESP_OK si el registro fue escrito correctamente.
  • ESP_ERR_INVALID_ARG si el puntero recibido es nulo.
  • ESP_FAIL si el archivo no está disponible o si ocurre un error de escritura.

Esta función recibe una estructura con una línea de texto ya preparada para almacenamiento y la agrega al archivo previamente abierto.

Su lógica es simple: validar que el registro y el archivo sean válidos, y delegar la escritura real al driver de la microSD.
