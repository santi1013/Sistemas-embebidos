Detailed Description

Implementación del módulo encargado de registrar datos en archivo.

Este módulo administra la apertura del archivo de log en la microSD y la escritura de los registros generados por el sistema. Su objetivo es separar la lógica de almacenamiento de la lógica de adquisición y procesamiento de datos del MPU6050.

La idea general es:

  • verificar que la microSD esté montada,
  • abrir el archivo de salida,
  • escribir el encabezado solo si el archivo está vacío,
  • y luego agregar cada nuevo registro que llegue desde las tareas del sistema.

◆ file_logger_init()

esp_err_t file_logger_init ( void )

Inicializa el archivo de log en la microSD.

Inicializa el módulo de escritura en archivo.

Esta función comprueba primero que la tarjeta SD esté montada. Después verifica si el archivo ya existe y si contiene información, usando stat(). Si el archivo está vacío, escribe el encabezado con los nombres de las columnas; si ya contiene datos, simplemente lo abre para seguir agregando registros al final.

Su lógica permite que el archivo conserve información previa y evita duplicar el encabezado en cada reinicio del sistema.

Returns
  • ESP_OK si el archivo quedó listo para escritura.
  • ESP_FAIL si la SD no está montada, si el archivo no pudo abrirse o si falló la escritura del encabezado.

◆ file_logger_write_record()

esp_err_t file_logger_write_record ( const log_record_t * record)

Escribe un registro formateado en el archivo de log.

Esta función recibe un registro ya convertido a texto y lo agrega al archivo abierto en la microSD. Antes de escribir, valida que el puntero al registro sea válido y que el archivo haya sido inicializado correctamente.

Su lógica es delegar la escritura real al driver de la SD, manteniendo este módulo como intermediario entre la aplicación y el sistema de archivos.

Parameters
recordPuntero al registro que contiene la línea de texto a guardar.
Returns
  • ESP_OK si el registro fue escrito correctamente.
  • ESP_ERR_INVALID_ARG si el puntero recibido es nulo.
  • ESP_FAIL si el archivo no está abierto o si ocurre un error de escritura.
